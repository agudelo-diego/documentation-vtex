POST get-contests-list-external

https://sbvtexapi.desarrolloaxa.com.co/public/api/v1/external/contests/list

Descripción

Este endpoint permite consultar una lista de concursos, que están asociados a un cliente específico, La petición incluye parámetros como el NIT del cliente, el estado del concurso y opciones de paginación para optimizar la respuesta, estos se enviaran en el body de la petición.

Parámetros

status_id

  • Requerido: Sí
  • Tipo: Entero
  • Validaciones:
  • Debe ser un número entero.
  • Descripción: Identificador del estado con el que se estimara que concursos retornar, estatusid:1 concursos vigentes por fechas, inicio de concurso menor al momento de ejecución y finalizacion superior a fecha de momento de ejecución. statusid:2 retornara cocncursos donde fecha de finalizacion ya paso, o sea concursos finalizados. status_id:3 retornara todos los concursos.

nit

  • Requerido: Sí
  • Tipo: Entero
  • Validaciones:
  • Debe ser un número entero.
  • Descripción: Número de Identificación Tributaria (NIT) del cliente o entidad a consultar.

page_size

  • Requerido: No (Opcional)
  • Tipo: Entero
  • Validaciones:
  • Debe ser un número entero.
  • Debe tener un valor mínimo de 1.
  • Su valor máximo permitido es 100.
  • Descripción: Define el número de registros que se retornarán por página en los resultados de la consulta.

page

  • Requerido: No (Opcional)
  • Tipo: Entero
  • Validaciones:
  • Debe ser un número entero.
  • Debe tener un valor mínimo de 1.
  • Descripción: Número de la página a consultar en el conjunto paginado de resultados.

sort_by

  • Requerido: No (Opcional)
  • Tipo: Cadena de texto
  • Validaciones:
  • Debe ser una cadena de texto.
  • Descripción: Campo por el cual se desea ordenar los resultados de la consulta.

order_desc

  • Requerido: No (Opcional)
  • Tipo: Booleano
  • Validaciones:
  • Debe ser un valor booleano (true o false).
  • Descripción: Define si el orden de los resultados será descendente (true) o ascendente (false). Si no se especifica, se utilizará un orden ascendente por defecto.

Posibles repuestas

Error de Validación: El servidor responde con un código HTTP 422 (Unprocessable Entity) si los datos enviados no cumplen con las reglas de validación. La respuesta incluye detalles de los errores en formato JSON.

Error del Servidor: En caso de un problema interno, el servidor retorna un código HTTP 500 (Internal Server Error).

Proceso Exitoso: El servidor devuelve un código de estado HTTP 200 (OK), indicando que la solicitud fue procesada correctamente. La respuesta incluye los datos de los concursos en formato JSON.

el api retornara un json con la siguiente estructura:

{
    "data": {
        "contests": [
            {
                "contest_id": 199,
                "contest_name": "¡Gana un Kia Picanto 0 km con Distribuciones AXA y Pfizer!",
                "contest_start_date": "2024-12-01",
                "contest_end_date": "2024-12-31",
                "status": "Participando",
                "is_multiple_opportunities_allowed": true,
                "is_draw_required": false,
                "general_quota": 50000,
                "quota_percentage": 0,
                "general_quota_completed": 0,
                "url_image": "http:\/\/127.0.0.1:8000\/api\/images\/contest_image199_0_1733843031.7387.jpg"
            }
        ],
        "total_contest": 2
    }
}

El API retornará un JSON con una propiedad data. Dentro de esta, estarán las propiedades contests, que es un arreglo con los concursos disponibles, y total_contest, que es un campo que permitirá saber cuántos concursos existen con los parámetros pasados. Esto se utilizará para el paginado.

Estructura de la Respuesta

  • data: Objeto principal que contiene la información sobre los concursos.

  • contests: Lista de concursos, donde cada concurso tiene los siguientes atributos:

    • contest_id: Identificador único del concurso (e.g., 199).
    • contest_name: Nombre del concurso (e.g., "¡Gana un Kia Picanto 0 km con Distribuciones AXA y Pfizer!").
    • conteststartdate: Fecha de inicio del concurso (e.g., "2024-12-01").
    • contestenddate: Fecha de finalización del concurso (e.g., "2024-12-31").
    • status: Estado actual del concurso (e.g., "Participando").
    • ismultipleopportunities_allowed: Booleano que indica si se permiten múltiples oportunidades de participación (e.g., true).
    • isdrawrequired: Booleano que indica si es necesario realizar un sorteo (e.g., false).
    • general_quota: Cuota general en compras segun condiciones(e.g., 50000).
    • quota_percentage: Porcentaje de cuota completado (e.g., 0).
    • generalquotacompleted: cumplimiento de cuota general (e.g., 0).
    • urlimage: URL de la imagen del concurso (e.g., "http://127.0.0.1:8000/api/images/contestimage19901733843031.7387.jpg").

  • total_contest: Número total de concursos (e.g., 2).

POST get-contests-details-external

https://sbvtexapi.desarrolloaxa.com.co/public/api/v1/external/contests/details

Descripción

Este endpoint permite consultar los detalles de un concurso en especifico. La petición incluye parámetros como el NIT del cliente y el id del concurso, estos se enviaran en el body de la petición.

Parámetros

nit

  • Requerido: Sí
  • Tipo: Entero
  • Validaciones:
  • Debe ser un número entero.
  • Descripción: Número de Identificación Tributaria (NIT) del cliente o entidad a consultar.

contest_id

  • Requerido: Sí
  • Tipo: Entero
  • Validaciones:
  • Debe ser un número entero.
  • Descripción: Identificador del concurso que se desea consultar.

Posibles repuestas

Error de Validación: El servidor responde con un código HTTP 422 (Unprocessable Entity) si los datos enviados no cumplen con las reglas de validación. La respuesta incluye detalles de los errores en formato JSON.

Error del Servidor: En caso de un problema interno, el servidor retorna un código HTTP 500 (Internal Server Error).

Proceso Exitoso: El servidor devuelve un código de estado HTTP 200 (OK), indicando que la solicitud fue procesada correctamente. La respuesta incluye los datos del concursos en formato JSON, con una propiedad data. Dentro de esta, estarán las propiedades del concurso.

Estructura de la Respuesta

  • data: Objeto principal que contiene la información sobre el concurso.
  • contest_id: Identificador único del concurso (e.g., 200).
  • contest_name: Nombre del concurso (e.g., "Concurso fecha vencidas").
  • contest_description: Descripción del concurso (e.g., "Concurso fecha vencidas").
  • conteststartdate: Fecha de inicio del concurso (e.g., "2024-11-01").
  • contestenddate: Fecha de finalización del concurso (e.g., "2024-11-30").
  • status: Estado actual del concurso (e.g., "Finalizado").
  • ismultipleopportunities_allowed: Booleano que indica si se permiten múltiples oportunidades de participación (e.g., true).
  • isdrawrequired: Booleano que indica si es necesario realizar un sorteo (e.g., false).
  • general_quota: Cuota general en compras según condiciones (e.g., 56234).
  • quota_percentage: Porcentaje de cuota completado (e.g., 55).
  • urlimage: URL de la imagen del concurso (e.g., "http:\/\/127.0.0.1:8000\/api\/images\/contestimage20001733947511.6897.png").
  • lastupdatedate: Fecha de la última actualización de datos (e.g., null).
  • opportunities_counter: Contador de oportunidades registradas (e.g., 0).
  • totalconditionscounter: Total de condiciones del concurso (e.g., 3).
  • missingconditionscounter: Contador de condiciones faltantes (e.g., 2).
  • generalquotacompleted: Monto completado de la cuota general (e.g., 31000).
  • tyc: Términos y condiciones del concurso (e.g., "tyc").
  • totalgeneralquota_completed: Monto total de la cuota general completado (e.g., "116000.00").
  • award_name: Nombre del premio del concurso (e.g., "CARRO KIA PICANTO").
  • remainingtonext_ticket: Monto restante para alcanzar el siguiente boleto (e.g., 25234).

contest_id

nit

  • Requerido: Sí
  • Tipo: Entero
  • Validaciones:

  • Descripción: Número de Identificación Tributaria (NIT) del cliente o entidad a consultar.

page_size

page

  • Requerido: No (Opcional)
  • Tipo: Entero
  • Validaciones:
  • Debe ser un número entero.
  • Debe tener un valor mínimo de 1.
  • Descripción: Número de la página a consultar en el conjunto paginado de resultados.

sort_by

  • Requerido: No (Opcional)
  • Tipo: Cadena de texto
  • Validaciones:
  • Debe ser una cadena de texto.

order_desc

  • Requerido: No (Opcional)
  • Tipo: Booleano
  • Validaciones:
  • Debe ser un valor booleano (true o false).
  • Descripción: Define si el orden de los resultados será descendente (true) o ascendente (false). Si no se especifica, se utilizará un orden ascendente por defecto.

Posibles repuestas

Estructura de la Respuesta

  • data: Objeto principal que contiene la información sobre las condiciones.

    • remaining_quota: Valor faltante para cumplir cuota de condición.
    • quota_value: Cuota configurada para la condición.
    • quota_percentage: Porcentaje completado para la cuota de la condición.
